## Hook Strategy

Each result video needs five hooks before the final script is written.

A hook is not just a clear opening. A hook should create dissonance:

1. Belief: name what the viewer probably believed before seeing the report.
2. Reversal: turn that belief over using their result.
3. Curiosity gap: open a question without answering it yet.
4. Bridge: point into the diagnosis.

The curiosity gap is the missing "why", not the result itself. You can reveal the flagged report state, because the viewer is already on the result page. Do not explain the likely causes inside the hook. The hook should make the viewer want the diagnosis, not replace it.

Curiosity gap test:

- Pass: the viewer thinks, "Hang on, if that is true, what is causing it?"
- Fail: the viewer thinks, "Okay, I know the answer now."

Do not list likely causes, first checks, or fixes in the hook. Save those for the diagnosis and fix-path sections.

Setup videos can be simpler. If the user is already mid-process and just needs instructions, orient them and tell them what to do next.
