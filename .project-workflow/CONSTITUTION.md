# Constitution

## Mission

- Keep JohnDetlefs.com product, content, lead-magnet, product-ad, and implementation work coordinated from the workspace root.
- Preserve clear ownership between the parent workspace repo and deployable child repos.
- Help Shopify merchants through practical coaching, reports, checklists, and useful creative/content assets.

## Repository Boundaries

- Parent repo: workspace/control repo for product memory, workflow state, roadmaps, lead magnets, guide/video packets, and `product-ads/`.
- `next/`: website application repo for deployable Next.js code and app-local workflow only.
- `email/`: email-related repo for deployable email code/assets and app-local workflow only.

## Product Direction

- Follow `PROJECT_CONTEXT.md` and `JOHNDETLEFS_PROGRAM_ROADMAP.md` for the current Shopify Growth Coaching direction.
- Treat older Fix-It Diagnostic references as legacy unless the owner explicitly asks to work on that funnel.

## Workflow Principles

- One task should have one workflow owner.
- Cross-repo work must list its primary repo, touched repos, and branch/PR state.
- Parent-owned work should not be tracked in a child repo tracker.
- Child repo trackers may exist only when their scope is explicitly app-local.
